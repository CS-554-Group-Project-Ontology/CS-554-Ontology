### General Organization 

* The npm library being used in order to connect to our Kafka Docker Image is kafka.js
(<https://kafka.js.org/docs/getting-started>). The syntax is in common javascript no es6 with the link provided.

* Current each Topic is an immutable log that is stored within kafka_broker on Railway.



### Available Scripts

**Runtime note:** This service is built and deployed on Node.js. Since I have been using Bun as an alternative runtime for faster local development, since it runs TypeScript natively without a build step and ouput to dist.I will provide a seperate pack.json that will be able to run this layer with no problems or issues. 

| Script | Runtime | Purpose |
|--------|---------|---------|
| `build` | Node (tsc) | Compile TypeScript to `dist/` |
| `start-node` | Node | Build, then run compiled JS on Node.js (production) |
| `start` | Bun | Run the server directly with Bun (no build step) |
| `start:bun` | Bun | Explicit alias for `start` |
| `test-xv2` | Bun | Run the X (Twitter) producer in isolation |
| `test-polymarket` | Bun | Run the Polymarket producer in isolation.

* Polymarket Gamma API

##### Producers and Consumers
*The two different API's go into two seperate events  are for X and Polymarket. The producers write to the given topics and the consumers subscribe to the given topic. A topic is a append only immutable log that holds that hold formatted event data in key value timestaps format. The number of paratitions is how many instances of the given topic that exist.
